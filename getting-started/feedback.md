# Submitting Feedback

Entry has a built-in feedback system so you can report bugs, request features, or suggest improvements — directly from within the app.

---

## How to Submit Feedback

1. Click the **blue chat icon** in the bottom-right corner of any page
2. Select a **category** — Bug Report, Feature Request, Improvement, or General
3. Enter a **subject** and **details** describing your feedback
4. Click **Submit Feedback**

The page you're currently viewing is automatically captured so the team knows where you were when submitting.

{% hint style="info" %}
The feedback button is available on every page in the app. You can submit feedback from anywhere.
{% endhint %}

---

## Tracking Your Feedback

Navigate to **Feedback** in the sidebar to see all your submitted feedback and its current status.

### Status Meanings

| Status          | What it means                                      |
| --------------- | -------------------------------------------------- |
| **Pending**      | Submitted — waiting for admin review               |
| **Reviewed**     | An admin has seen your feedback                    |
| **In Progress**  | Your feedback is being worked on                   |
| **Actioned**     | Done — the change has been made                    |
| **Dismissed**    | Reviewed but won't be actioned (with explanation)  |

---

## Admin Responses

When an admin updates your feedback, you'll see their response directly on the feedback item. This includes:

* A **written response** explaining what action was taken
* **Who** actioned it and **when**

{% hint style="success" %}
You can only see your own feedback submissions. Other users' feedback is not visible to you — only admins can see all submissions across the organisation.
{% endhint %}

---

## Filtering

Use the filter buttons at the top of the Feedback page to view items by status — for example, show only **Pending** items or only **Actioned** items.
